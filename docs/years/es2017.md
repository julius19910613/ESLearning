# ES2017

ES2017 的主角是 `async/await`。它没有取代 Promise，而是把 Promise 驱动的异步流程写得更像同步代码。

## async / await

```js
function wait(ms) {
  return new Promise(resolve => setTimeout(resolve, ms));
}

async function run() {
  await wait(50);
  return 'ready';
}

run().then(console.log); // ready
```

- `async function` 总是返回 Promise。
- `await` 只能在 async 函数内部使用，直到 ES2022 才有 top-level await。
- 对彼此独立的任务，优先考虑 `Promise.all()` 并发，而不是连续 `await`。

## Object.values / Object.entries

```js
const scores = { js: 92, css: 85, html: 88 };

console.log(Object.values(scores)); // [92, 85, 88]
console.log(Object.entries(scores)); // [['js', 92], ['css', 85], ['html', 88]]
```

这两个 API 让对象更容易进入数组处理链路，例如 `map`、`filter`、`sort`。

## String Padding

```js
const id = '42';

console.log(id.padStart(5, '0')); // 00042
console.log(id.padEnd(5, '.')); // 42...
```

- 常用于展示层格式化。
- 不建议把它当作严肃的数据校验逻辑。

## Trailing Commas

```js
const config = {
  host: 'localhost',
  port: 3000,
};

function sum(
  a,
  b,
) {
  return a + b;
}
```

尾随逗号让多行编辑的 diff 更干净，尤其是在对象、数组、参数列表不断增减时。

## Shared Memory / Atomics 概览

```js
const buffer = new SharedArrayBuffer(Int32Array.BYTES_PER_ELEMENT * 2);
const view = new Int32Array(buffer);

Atomics.add(view, 0, 1);
console.log(view[0]); // 1
```

- `SharedArrayBuffer` 允许多个执行上下文共享同一块内存。
- `Atomics` 提供原子读写与同步原语。
- 这类能力主要用于 Worker、并发计算和底层性能场景。
