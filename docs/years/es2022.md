# ES2022

ES2022 让类、模块与错误对象都更成熟，也把几个长期常用的小工具正式补齐。

## Class Fields / Private Fields / Private Methods / Static Blocks

```js
class Counter {
  static defaultStep = 1;
  static registry = [];

  #value = 0;

  static {
    this.registry.push('Counter initialized');
  }

  #format() {
    return `value=${this.#value}`;
  }

  inc(step = Counter.defaultStep) {
    this.#value += step;
    return this.#format();
  }
}

const counter = new Counter();
console.log(counter.inc()); // value=1
console.log(Counter.registry);
```

- class fields 让实例状态更直观。
- `#private` 是语言级私有，不是命名约定。
- `static {}` 适合类级初始化逻辑。

## Top-level await

```js
const config = await Promise.resolve({ lang: 'zh-CN' });
console.log(config.lang);
```

这让模块初始化流程更自然，尤其是在配置加载、资源预热和 ESM-only 工具链里。

<div class="support-note">
  <p>兼容性提示：top-level await 只适用于模块上下文。它会影响模块依赖图的加载顺序，使用时要避免把大量网络等待放到入口模块里。</p>
</div>

## Error cause

```js
try {
  try {
    JSON.parse('{ bad json }');
  } catch (error) {
    throw new Error('配置解析失败', { cause: error });
  }
} catch (error) {
  console.log(error.message);
  console.log(error.cause);
}
```

当错误需要逐层包装时，`cause` 能比拼接字符串更清晰地保留原始上下文。

## Object.hasOwn

```js
const user = Object.create({ inherited: true });
user.name = 'Lin';

console.log(Object.hasOwn(user, 'name')); // true
console.log(Object.hasOwn(user, 'inherited')); // false
```

它比 `obj.hasOwnProperty(...)` 更稳，因为不依赖对象自身原型链。

## Array.prototype.at / String.prototype.at

```js
const arr = ['a', 'b', 'c'];
const text = 'hello';

console.log(arr.at(-1)); // c
console.log(text.at(-2)); // l
```

`.at()` 的核心价值是支持负索引，读最后一项时更简洁。

## RegExp Match Indices

```js
const match = /world/d.exec('hello world');

console.log(match.indices[0]); // [6, 11]
```

加上 `d` flag 后，正则结果会附带每个匹配片段的起止下标，做高亮、语法分析和文本编辑器功能时很有用。
