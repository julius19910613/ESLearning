# ES2020

ES2020 是现代业务代码里最常出现的语法升级点之一。`?.` 和 `??` 几乎已经成为日常写法。

## Optional Chaining

```js
const user = {
  profile: {
    name: 'Lin'
  }
};

console.log(user?.profile?.name); // Lin
console.log(user?.settings?.theme); // undefined
```

它能避免成串的存在性判断，但只会在 `null` / `undefined` 时短路。

## Nullish Coalescing

```js
console.log(0 ?? 100); // 0
console.log('' ?? 'fallback'); // ''
console.log(null ?? 'fallback'); // fallback
```

当你只想处理“空值”，不要再用 `||` 误伤 `0`、`false`、空字符串。

## BigInt

```js
const maxSafe = 9007199254740991n;
const next = maxSafe + 2n;

console.log(next); // 9007199254740993n
```

- `BigInt` 适合超出 `Number.MAX_SAFE_INTEGER` 的整数运算。
- 不能与普通 `number` 直接混算，必须显式转换。

## Promise.allSettled

```js
const tasks = [
  Promise.resolve('ok'),
  Promise.reject(new Error('bad'))
];

Promise.allSettled(tasks).then(console.log);
```

当你需要“全部任务都结束后再看结果”，而不是遇到一个失败就整体 reject，`allSettled()` 很合适。

## globalThis

```js
globalThis.appVersion = '2020';
console.log(globalThis.appVersion);
```

它提供统一的全局对象入口，避免在不同环境里写 `window` / `global` / `self` 分支。

## Dynamic import

```js
async function loadMath() {
  const mod = await import('./math.js');
  return mod;
}
```

- 适合懒加载、按需分包、条件加载。
- 与 bundler 配合时，通常会变成独立 chunk。

## String.prototype.matchAll

```js
const text = 'x=1 y=2';
const matches = [...text.matchAll(/(\w)=(\d)/g)];

console.log(matches.map(match => match[0])); // ['x=1', 'y=2']
```

`matchAll()` 让“拿到全部匹配及其捕获组”变得更自然，不必手写 `exec()` 循环。
