# ES2023

ES2023 重点强化数组的“不可变操作体验”。如果你经常写状态更新逻辑，这一页会很有价值。

## Array.prototype.findLast / findLastIndex

```js
const logs = [
  { level: 'info', message: 'start' },
  { level: 'warn', message: 'slow' },
  { level: 'warn', message: 'retry' }
];

console.log(logs.findLast(item => item.level === 'warn'));
console.log(logs.findLastIndex(item => item.level === 'warn')); // 2
```

它们从数组尾部开始搜索，比手写反向循环更直接。

## toSorted / toReversed / toSpliced / with

```js
const arr = [3, 1, 2];

console.log(arr.toSorted((a, b) => a - b)); // [1, 2, 3]
console.log(arr.toReversed()); // [2, 1, 3]
console.log(arr.toSpliced(1, 1, 99)); // [3, 99, 2]
console.log(arr.with(0, 42)); // [42, 1, 2]
console.log(arr); // [3, 1, 2]
```

- 这组 API 的共同点是“不改原数组”。
- 它们对 React、Vue、状态机、不可变数据流都很友好。

## Hashbang Grammar

```js
#!/usr/bin/env node
console.log('run as script');
```

这让脚本文件顶部的 shebang 在语言层面被识别，而不是只依赖宿主忽略。

## Symbols as WeakMap Keys 概览

```js
const key = Symbol('session');
const map = new WeakMap();

// 某些较新的引擎会逐步扩展可作为弱映射 key 的值域。
// 在真正依赖之前，请先验证目标运行时行为。
console.log(typeof key); // symbol
console.log(map instanceof WeakMap); // true
```

- 这类能力与垃圾回收语义直接相关。
- 不同引擎、不同版本的落地节奏可能不一致。
- 如果你需要跨环境稳定行为，仍然优先使用对象作为 `WeakMap` key。
