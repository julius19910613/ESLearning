# ES2025

ES2025 代表的是“更现代的数据流与标准库增强”。这一页里的多项能力在语义上很自然，但请默认它们需要非常新的运行时。

## Iterator Helpers

```js
function* numbers() {
  yield 1;
  yield 2;
  yield 3;
  yield 4;
}

const result = numbers()
  .map(n => n * 2)
  .filter(n => n > 4)
  .take(2)
  .toArray();

console.log(result); // [6, 8]
```

- 把数组风格的链式处理带到 iterator。
- 适合惰性数据流、超大序列、按需消费。

## Set Methods

```js
const a = new Set([1, 2, 3]);
const b = new Set([3, 4, 5]);

console.log([...a.union(b)]); // [1, 2, 3, 4, 5]
console.log([...a.intersection(b)]); // [3]
console.log(a.isSubsetOf(new Set([1, 2, 3, 4]))); // true
```

常见新方法包括：

- `union()`
- `intersection()`
- `difference()`
- `symmetricDifference()`
- `isSubsetOf()`
- `isSupersetOf()`
- `isDisjointFrom()`

## Promise.try

```js
Promise.try(() => JSON.parse('{"ok":true}'))
  .then(result => console.log(result.ok))
  .catch(console.error);
```

它把“把同步或异步起点都收进 Promise 流程”这件事标准化了。

## RegExp.escape

```js
const input = '(hello)+';
const safe = RegExp.escape(input);
const re = new RegExp(`^${safe}$`);

console.log(re.test('(hello)+')); // true
```

当你要把用户输入拼进正则时，`RegExp.escape()` 比手写转义更可靠。

## RegExp Pattern Modifiers / Duplicate Named Capture Groups

```js
// 依赖非常新的正则实现，示意为主：
// /(?i:hello) world/
// /(?<value>\d+)-(?<value>\w+)/
console.log('Check runtime support before using advanced regex modifiers.');
```

- pattern modifiers 让局部修饰符更细粒度。
- duplicate named capture groups 让复杂模式复用命名更自然。
- 这类特性适合高级文本处理，不适合盲目追新。

## JSON Modules / Import Attributes

```js
import config from './config.json' with { type: 'json' };

console.log(config.name);
```

- JSON module 让 JSON 成为正式模块输入。
- import attributes 用于声明资源类型等加载意图。
- 在 bundler、Node.js、浏览器三端的支持节奏未必完全同步。

## Float16Array 概览

```js
if (typeof Float16Array === 'function') {
  const half = new Float16Array([1.5, 2.25]);
  console.log(half.length);
}
```

`Float16Array` 面向半精度浮点数据，常见于图形、ML、WebGPU 或高密度数值缓冲区场景。
