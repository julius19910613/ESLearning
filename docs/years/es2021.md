# ES2021

ES2021 主要是对现有能力的“顺手增强”，但 `Promise.any()` 和逻辑赋值在业务代码里很有存在感。

## Logical Assignment Operators

```js
const options = { retry: 0, timeout: null };

options.retry ||= 3;
options.timeout ??= 5000;

console.log(options); // { retry: 3, timeout: 5000 }
```

- `||=`：左侧为 falsy 时赋值。
- `&&=`：左侧为 truthy 时赋值。
- `??=`：左侧为 nullish 时赋值。

## Numeric Separators

```js
const bytes = 1_048_576;
const hex = 0xff_ff;

console.log(bytes, hex);
```

数字分隔符只影响可读性，不影响数值本身，适合金额、字节数、时间戳分组。

## Promise.any 与 AggregateError

```js
const mirrors = [
  Promise.reject(new Error('A down')),
  Promise.resolve('B ok'),
  Promise.resolve('C ok')
];

Promise.any(mirrors).then(console.log); // B ok
```

当全部候选都失败时，`Promise.any()` 会抛出 `AggregateError`：

```js
Promise.any([Promise.reject('x'), Promise.reject('y')]).catch(error => {
  console.log(error instanceof AggregateError); // true
  console.log(error.errors); // ['x', 'y']
});
```

## String.prototype.replaceAll

```js
const slug = 'es 2021 notes'.replaceAll(' ', '-');
console.log(slug); // es-2021-notes
```

它让“替换全部出现位置”的意图比正则全局替换更直观。

## WeakRefs 概览

```js
let obj = { value: 42 };
const ref = new WeakRef(obj);

console.log(ref.deref()?.value); // 42
obj = null;
```

- `WeakRef` 允许你拿到一个不会阻止 GC 的弱引用。
- 它适合缓存、底层资源管理等少数场景。
- 绝大多数业务代码不应该依赖对象何时被 GC。
