# ES2015

ES2015 是现代 JavaScript 的分水岭。很多今天被视为“默认写法”的能力，都是从这一年开始进入主流。

## let / const

```js
let count = 0;
count += 1;

const settings = { theme: 'light' };
settings.theme = 'dark';

console.log(count); // 1
console.log(settings.theme); // dark
```

- `let` 提供块级作用域。
- `const` 禁止重新绑定，但不冻结对象内容。
- 两者都存在 temporal dead zone，不能像 `var` 一样提前随意读取。

## Arrow Functions 与 Template Literals

```js
const prices = [10, 20, 30];
const withTax = prices.map(price => price * 1.13);
const report = `含税价格：${withTax.join(', ')}`;

console.log(report);
```

- arrow function 会捕获外层 `this`，适合短回调。
- 不要把箭头函数当构造函数使用。
- template literals 更适合多行文本和字符串插值。

## Destructuring, Default Parameters, Rest / Spread

```js
function createUser({ name, role = 'reader' } = {}) {
  return { name, role };
}

const user = createUser({ name: 'Lin' });
const nums = [1, 2, 3];
const clone = [...nums, 4];
const { name, role } = user;

console.log(name, role); // Lin reader
console.log(clone); // [1, 2, 3, 4]
```

- 解构适合取对象字段、交换变量、声明默认值。
- rest 用于“收集剩余参数”。
- spread 用于“展开已有内容”，但对象和数组都只是浅拷贝。

## Classes

```js
class Counter {
  constructor(start = 0) {
    this.value = start;
  }

  inc() {
    this.value += 1;
    return this.value;
  }
}

const c = new Counter(2);
console.log(c.inc()); // 3
```

- `class` 是基于原型系统的语法糖，不是另一套对象模型。
- 方法默认定义在原型上。
- 若只需要简单对象组合，不必强行 class 化。

## Modules

`math.js`

```js
export function add(a, b) {
  return a + b;
}
```

`main.js`

```js
import { add } from './math.js';

console.log(add(2, 3)); // 5
```

模块让依赖边界更清晰，也为后来的 tree-shaking、dynamic import 和 top-level await 打下基础。

## Promises

```js
function wait(ms) {
  return new Promise(resolve => {
    setTimeout(() => resolve(`done in ${ms}ms`), ms);
  });
}

wait(100).then(console.log);
```

- Promise 统一了异步完成值与错误传播。
- `then` 返回新 Promise，因此可以链式组合。
- 业务代码里常见错误是漏掉 `catch`，或把并发写成串行。

## Generators

```js
function* idGenerator() {
  let id = 1;
  while (true) {
    yield id++;
  }
}

const ids = idGenerator();
console.log(ids.next().value); // 1
console.log(ids.next().value); // 2
```

- generator 让函数可以暂停与恢复。
- 它曾大量用于异步流程库；今天更多用于自定义 iterable 或惰性序列。

## Map / Set

```js
const map = new Map([
  ['name', 'ES2015'],
  ['type', 'standard']
]);

const set = new Set([1, 2, 2, 3]);

console.log(map.get('name')); // ES2015
console.log([...set]); // [1, 2, 3]
```

- `Map` 适合真正的键值映射，不会与原型链属性混淆。
- `Set` 适合去重与成员检测。

## Symbol

```js
const id = Symbol('id');
const user = {
  [id]: 42,
  name: 'Ada'
};

console.log(user[id]); // 42
console.log(Object.keys(user)); // ['name']
```

`Symbol` 常用于定义不会意外重名的属性键，也被用于实现语言协议，例如 `Symbol.iterator`。

## Proxy 与 Reflect

```js
const target = { count: 0 };

const proxy = new Proxy(target, {
  get(obj, key, receiver) {
    console.log('read ->', key);
    return Reflect.get(obj, key, receiver);
  },
  set(obj, key, value, receiver) {
    console.log('write ->', key, value);
    return Reflect.set(obj, key, value, receiver);
  }
});

proxy.count += 1;
console.log(proxy.count);
```

- `Proxy` 可以拦截对象操作。
- `Reflect` 提供与这些底层操作对应的默认行为。
- 这组能力很强，但也可能让调试、性能与类型推断变复杂，适合框架、响应式系统、元编程场景。
