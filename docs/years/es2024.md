# ES2024

ES2024 既有高频的 `groupBy()`，也有偏底层的内存、原子操作和正则能力。项目里通常是“少量高频 + 多项按需使用”。

## Object.groupBy / Map.groupBy

```js
const posts = [
  { id: 1, status: 'draft' },
  { id: 2, status: 'published' },
  { id: 3, status: 'draft' }
];

const groupedObject = Object.groupBy(posts, post => post.status);
const groupedMap = Map.groupBy(posts, post => post.status);

console.log(groupedObject.draft.length); // 2
console.log(groupedMap.get('published').length); // 1
```

- `Object.groupBy()` 输出普通对象。
- `Map.groupBy()` 输出 `Map`，适合 key 不只是字符串时使用。

## Promise.withResolvers

```js
const { promise, resolve, reject } = Promise.withResolvers();

setTimeout(() => resolve('done'), 50);
promise.then(console.log);
```

它把“创建 promise + 暴露 resolve/reject”的常见样板收成一个标准 API。

## ArrayBuffer Resize / Transfer 概览

```js
const buffer = new ArrayBuffer(8, { maxByteLength: 16 });
const resized = buffer.resize ? (buffer.resize(12), buffer) : buffer;

console.log(resized.byteLength);
```

- 可调整大小的 `ArrayBuffer` 让底层二进制处理更灵活。
- `transfer()` / `transferToFixedLength()` 则面向所有权转移与缓冲区管理。
- 这些能力主要服务于高性能 I/O、编解码和底层库作者。

## RegExp `v` flag / String Properties 概览

```js
// 下面这种写法需要非常新的正则实现：

console.log('Use v-flag aware runtime before relying on this.');
```

- `v` flag 为 Unicode 集合与更复杂的字符集表达提供更一致的语义。
- string properties 进一步增强了对复杂 Unicode 文本的描述能力。
- 如果你的业务牵涉 emoji、国际化搜索或文本规范化，这一方向值得关注。

## Atomics.waitAsync 概览

```js
async function waitExample() {
  const sab = new SharedArrayBuffer(4);
  const view = new Int32Array(sab);

  if (typeof Atomics.waitAsync === 'function') {
    const result = Atomics.waitAsync(view, 0, 0);
    console.log(await result.value);
  }
}
```

`Atomics.waitAsync()` 面向更现代的异步等待场景，尤其是不能阻塞主线程的执行环境。
