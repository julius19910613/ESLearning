# ES2019

ES2019 没有改变语言风格，但新增了一组非常顺手的“修整型 API”。

## Array.prototype.flat / flatMap

```js
const nested = [1, [2, [3]]];
console.log(nested.flat(2)); // [1, 2, 3]

const words = ['es', '2019'];
console.log(words.flatMap(word => [word, word.toUpperCase()]));
```

- `flat(depth)` 压平数组。
- `flatMap()` 等于先 `map()` 再 `flat(1)`，很适合“一进多出”的转换。

## Object.fromEntries

```js
const entries = [
  ['name', 'ES2019'],
  ['kind', 'update']
];

console.log(Object.fromEntries(entries));
```

它与 `Object.entries()` 互为镜像，让对象 <-> entries 的转换链路闭环。

## Optional Catch Binding

```js
try {
  JSON.parse('{ invalid json }');
} catch {
  console.log('parse failed');
}
```

如果你根本不需要错误对象，可以省略 `catch (error)` 里的变量名。

## Stable sort

```js
const items = [
  { group: 'a', score: 2 },
  { group: 'a', score: 1 },
  { group: 'b', score: 1 }
];

items.sort((a, b) => a.score - b.score);
console.log(items);
```

稳定排序意味着：比较结果相等时，原有相对顺序会保留。对多关键字排序链路尤其重要。

## trimStart / trimEnd

```js
const raw = '   hello world   ';

console.log(raw.trimStart()); // 'hello world   '
console.log(raw.trimEnd()); // '   hello world'
```

- 适合做输入清洗和文本预处理。
- 它们比历史别名 `trimLeft` / `trimRight` 更符合标准命名。
